## 2026-03-18 开发日志

### 一、配置模块 viper 重构

#### 1. 重构背景
原有配置管理直接写在 `main.go` 中，存在以下问题：
- 配置解析逻辑与业务逻辑耦合
- 仅支持 YAML 格式，不支持环境变量
- 需要手动处理默认值和命令行参数合并
- 缺乏独立的配置模块，难以测试

#### 2. 重构方案
引入 [spf13/viper](https://github.com/spf13/viper) 配置库，实现专业的配置管理：

**配置加载优先级**（从高到低）：
1. 命令行参数（最高优先级）
2. 环境变量（`KVCACHE_*` 前缀）
3. 配置文件（支持 YAML/JSON/TOML）
4. 默认值

#### 3. 新增文件
- `internal/config/config.go` - 配置模块核心实现
- `internal/config/config_test.go` - 配置模块单元测试

#### 4. 核心设计

**Config 结构体**：
```go
type Config struct {
    Address     string `mapstructure:"address"`
    DataDir     string `mapstructure:"data-dir"`
    NoPersist   bool   `mapstructure:"no-persist"`
    RewriteSize int64  `mapstructure:"rewrite-size"`
    MaxMemory   int64  `mapstructure:"max-memory"`
    EvictionPolicy string `mapstructure:"eviction-policy"`
}
```

**Loader 加载器**：
- `NewLoader()` - 创建加载器，设置默认值和环境变量支持
- `Load()` - 加载配置（配置文件 → 环境变量 → 默认值）
- `SetConfigFile()` - 设置配置文件路径

**默认值管理**：
- 统一在 `DefaultConfig()` 中定义默认值
- `NewLoader()` 从 `DefaultConfig()` 同步到 viper
- 避免重复定义，确保单一数据源

#### 5. 使用示例

```bash
# 命令行参数
./kv-cache.exe --config ./config.yaml --data ./mydata --maxmemory 104857600

# 环境变量
set KVCACHE_DATA_DIR=./mydata
set KVCACHE_MAXMEMORY=104857600
./kv-cache.exe

# 配置文件 (config.yaml)
data-dir: "./data"
max-memory: 104857600
eviction-policy: "lru"
```

#### 6. 测试覆盖
- `TestDefaultConfig` - 验证默认值正确性
- `TestConfigValidate` - 验证配置校验（淘汰策略有效性）
- `TestLoadFromFile` - 验证从文件加载配置
- `TestLoaderWithFlags` - 验证命令行参数绑定
