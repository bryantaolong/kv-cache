## 关键设计决策

### 1. 三层架构
```
命令层 (cmd_*.go)  → 存储层 (store.go) → 数据结构层 (datatype.go)
- 业务逻辑          - 并发安全          - 纯内存操作
- 类型检查          - 过期管理          - 算法实现
- 过期保留          - 锁管理
```

### 2. 类型安全
- 使用 `DataType` 枚举标记 Value 类型
- 类型断言时先检查 `DataType`，避免 panic
- 自定义 `ErrWrongType` 错误，符合 Redis 行为

### 3. 过期时间管理
- 所有修改操作保留原有过期时间
- 惰性删除：访问时检查过期
- 辅助方法统一处理 TTL 计算

## 遇到的问题与解决

### 1. map 引用问题
**问题**: `HGetAll` 直接返回内部 map，外部修改会影响存储。  
**解决**: 返回副本（深拷贝）。

### 2. ZSet 排序维护
**问题**: ZSet 需要同时维护 dict 和 sorted 数组，更新时保持一致性。  
**解决**: 
- 更新时先从 sorted 删除旧位置
- 二分查找插入新位置（保持有序）
- score 相同时按 member 字典序排列

### 3. 代码复用
**问题**: 各命令中重复出现类型检查、过期时间处理逻辑。  
**解决**: 提取辅助方法 `getOrCreateHash`, `getHash` 等。
