## 已完成工作

### 1. 过期键主动清理 (GC模块) ✅

**文件**: `internal/storage/gc.go` (新增, 88行)

- 实现后台 Goroutine 定期扫描清理过期键
- 采用两阶段清理策略：读锁收集过期键 → 写锁删除，减少锁竞争
- 支持动态启动/停止 (`Start()` / `Stop()`)
- 默认每分钟执行一次清理

```go
gc := NewGC(dataFunc)
gc.Start(time.Minute)  // 启动 GC
defer gc.Stop()        // 停止 GC
```

### 2. 内存上限与淘汰策略 (Evictor模块) ✅

**文件**: `internal/storage/evict.go` (新增, 196行)

- 支持 5 种淘汰策略：
  - `noeviction` - 不淘汰（默认）
  - `allkeys-lru` - 所有键 LRU 淘汰
  - `volatile-lru` - 仅淘汰带过期时间的键
  - `allkeys-random` - 所有键随机淘汰
  - `volatile-random` - 仅淘汰带过期时间的键（随机）

- 内存估算覆盖所有数据类型（String/Hash/List/Set/ZSet）
- 淘汰目标：将内存使用降至上限的 75% 以下

**新增命令行参数**:
```bash
./kv-cache -maxmemory 268435456 -maxmemory-policy allkeys-lru
```

### 3. List LPush 性能优化 ✅

**文件**: `internal/storage/types/list.go`

**优化前**: O(n*m) - 每次插入都复制整个 slice  
**优化后**: O(n) - 一次性构建，只需一次复制

```go
// 优化前：循环中多次 append，每次可能触发扩容
for _, val := range vals {
    l.Items = append([]string{val}, l.Items...)
}

// 优化后：预分配空间，一次性构建
newItems := make([]string, len(vals), len(vals)+len(l.Items))
copy(newItems, vals)
l.Items = append(newItems, l.Items...)
```

## 技术要点

1. **锁优化**: GC 和 Evictor 都使用 RLock 进行数据遍历，减少写锁持有时间
2. **内存估算**: 采用简化算法，只计算 value 大小，不包含 Go 运行时开销
3. **LRU 简化**: 当前使用随机采样（随机选3个淘汰1个），非完整 LRU 实现
